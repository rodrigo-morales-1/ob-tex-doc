* DONE Find a way to remove the PDF file while it is opened in Okular in Windows 11
 
In GNU/Linux, there is no restriction when trying to remove a PDF file that is opened by Okular, after the new PDF is generated, the running instance of Okular shows the new PDF file. In Windows 11, the system doesn't allow Emacs to remove a file that is currently being shown by Okular, so it is not possible to execute code block because it is not possible to remove the previous PDF file when trying to clean the directory.

Perhaps, a solution in Windows 11 would be to clean all files except the PDF file, because that's the only file that is opened by a process and is causing the issue.
